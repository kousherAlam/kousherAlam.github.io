My Strategy for developing a project from ground to production

When i start a project, i begain with the business. I think if i can understand the business properly then it will made working with project much easier.

So i discuss the business multiple time with client and then start to develop the prototype.

If we made the prototype ready, then we discuss about the workflow again. In this time we found many unnecessary freature or very important missing feature

Now time to working with the application automation and architecture. It take some time, because i think this is the most important part of an application, yes i know it's common to chnage application architecture but by creating an loosly couple architecture we can make sure to easily any part out or in.

Now all the pices on place, it's time to make some todo and time estimation

By this time application architecture, todo, time-estimation everything is ready. Now it's time to ready our first prototype version

When prototype version 1 ready, now need to discuss and check all business are ok or not?

If everything is ok then deploy it for Quality Testing , and Release. If not, then to the process from requirement to prototype version.
